# 📝 ブログ執筆の追求ポイント：読者を惹きつける技術記事の作り方

ユーザー様が本プロジェクトをブログとして公開する際、より深みを持たせるための「追求テーマ」を提案します。

## 1. 「なぜJSで物理演算なのか？」の対比

単に「作ってみた」だけでなく、以下の対比を強調すると技術ブログとしての価値が高まります。

- **直感性 (JS)**: ブラウザさえあれば動く。手軽に開発できる。
- **限界 (JS)**: メインスレッドを占有するため、演算負荷が上がるとUIがカクつく。
- **期待 (Rust/Wasm)**: 「型」による堅牢性と、ハードウェアに近い速度。
  - *追求ポイント*: Rust移植の際に「JSのこの部分がボトルネックだった」という実体験を記す。

## 2. アルゴリズムの「視覚的」解説案

Stable Fluids は数学的には難解（ナビエ・ストークス方程式）ですが、ブログでは以下のように例えると読者に伝わりやすくなります。

- **拡散**: 「お風呂にインクを垂らした時の広がり」
- **移流**: 「川に浮かべた葉っぱが、水の流れに乗って運ばれる様子」
- **投影**: 「水はどこまでも縮まない（非圧縮性）ので、溜まった圧力を逃がしてあげる作業」

## 3. 「石（障害物）」がもたらすストーリー

障害物は単なる「壁」ではありません。

- **カルマン渦の再現**: 石の後ろにできる渦巻き。これが表現できると、シミュレーターとしてのリアリティが一気に跳ね上がります。
- **ユーザー体験**: 「流れるものを止める」という根源的な楽しさを語る。

## 4. 失敗談こそが最強の武器

本プロジェクトの開発中に起きた「真っ黒な画面」や「タイポによる流体の偏り」は、素晴らしいコンテンツです。

- **ImageData の罠**: 毎フレーム Canvas を作ると遅くなる。
- **Alpha チャンネルの罠**: 密度 0 だと透明になり、背景の黒に溶け込んで「何も映っていない」ように見えてしまう。
  - *追求ポイント*: 「動かない」と思った時、どこを最初に疑ったか？（今回は描画ロジックの透明度だった）
